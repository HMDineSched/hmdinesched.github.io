Explaining CSS with CSS

A meta journey into Cascading Style Sheets

The Relationship: HTML & CSS

Think of HTML as the **scaffolding** of a building. It provides the structure, the rooms, the walls, and the foundational elements.

HTML: Scaffolding

Provides the content, the elements, and the hierarchical structure of your web page. Without HTML, there's nothing to style!

<h1>Hello</h1><p>World</p>

CSS, on the other hand, provides the **meaning through styling**. It's the paint, the wallpaper, the furniture, the lighting, and the overall aesthetic that makes the building appealing and functional.

CSS: Meaning through Styling

Takes the raw HTML elements and applies visual properties like colors, fonts, spacing, and layout, transforming plain content into an engaging experience.

h1 {
    color: blue;
    font-family: sans-serif;
}
p {
    margin-top: 10px;
}

Visual Metaphors in Action

The "Shadowed" Definition: box-shadow

To literally "shadow" a definition, we can use the box-shadow CSS property. It adds a shadow effect around an element's frame, making it visually distinct and emphasizing its importance, much like a spotlight.

What is CSS?

CSS (Cascading Style Sheets) is a stylesheet language used to describe the presentation of a document written in HTML or XML (including XML dialects such as SVG, MathML, or XHTML). CSS describes how elements should be rendered on screen, on paper, in speech, or on other media.

Notice how this box literally has a shadow, emphasizing its definition.

The box-shadow property takes values for horizontal offset, vertical offset, blur radius, spread radius, and color. Experiment with them!

"Flying Off into Space": position: absolute

The position property, especially when set to absolute, allows elements to be removed from the normal document flow. They are then positioned relative to their closest *positioned* ancestor (an ancestor with a position other than static). This can make them "fly" independently!

This is the container for our flying box. It has position: relative;

I'm an absolutely positioned box!

My container provides the context for my flight.

When you hover over the red box above, it will demonstrate position: absolute by animating "off into space," detached from the normal layout flow.

"Reusable Ingredients": CSS Custom Properties (--var)

CSS Custom Properties, often called CSS Variables, allow you to define values that can be reused throughout your stylesheets. This makes your CSS more maintainable, readable, and easier to update, like having a consistent color palette or typography settings.

Our primary color for headings and header background is defined as --primary-color: #3498db;

Our secondary color for highlights is defined as --secondary-color: #2ecc71;

We use var(--primary-color) to apply the primary color to elements like this header, and var(--secondary-color) for elements like this box's background.

If we change --primary-color in one place, every element using var(--primary-color) will update!

In our stylesheet, we defined variables like --primary-color and --secondary-color. Look at the CSS for header and .variables-demo to see them in action!

"Adapting to the Environment": Media Queries

Media queries are a fundamental part of responsive web design. They allow you to apply different CSS styles based on device characteristics, such as screen width, height, resolution, and orientation. This means your website can "respond" and adapt its layout and presentation to best fit the user's screen.

Resize your browser window to see this content change!

This is responsive design!

As you resize your browser window (or view on different devices), the message inside the box above will change, demonstrating how media queries can reveal or alter content based on screen size.

  • Small Screens (e.g., mobile): Text might be red and say "You're on a small screen!"
  • Medium Screens (e.g., tablet): Text might be orange and say "You're on a medium screen!"
  • Large Screens (e.g., desktop): Text might be green and say "You're on a large screen!"
